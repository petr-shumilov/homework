ОПИСАНИЕ

1) Идея. Зачем это делалось?
Стояла задача рассылки новостей на e-mail/sms клиентам небольшой компании. Заниматься этим предстояло маркетологу.
Дабы упростить жизнь, была придумана схема:
  
  GUI(web) ==> WebSocket --------> Server( >10 clients at same time) ==> Worker
  
Протокол websocket был выбран, чтобы сервер мог моментално оповещать пользоватлей о прогрессе отправки или ошибках
Воркер выполняет одно задание(напрмер, отсылает смс по списку пользователя) параллельно. 
Все пользователи в системе равноправны, каждый может управлять воркером

2) Как пользоваться
  !!! работает толко на Python 3.x
  
  1. запускаем сервер ./server.py
  2. открываем GUI/index.html, браузер должен поддерживать протокол WebSocket
  3. если видем "System: connection has been opened", сервер начнёт отслылать каждые 10 сек статус воркера
     В GUI доступны следующие команды:
     -- run - запускает воркер
     -- stop - останавливает воркер
     -- kill - убивает воркер, далее работа с сервером бессмысленна
     В консоле сервер доступные следующие команды:
	-t - отправляет всем клиентам "Hello"
